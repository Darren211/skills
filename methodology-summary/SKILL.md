# 方法论总结

> **核心框架**：黄金圈法则（Golden Circle）—— 从 **Why** 出发，经由 **How**，最终到达 **What**。
>
> 每条方法论都必须回答三个问题：
> 1. **Why** — 为什么要这样做？（信念、目的、意义）
> 2. **How** — 怎么做？（原则、策略、方法）
> 3. **What** — 做了什么 / 具体行动是什么？（行为、成果、实践）

## 触发条件

当用户请求涉及以下关键词时触发：
- `方法论总结`、`经验沉淀`、`复盘总结`
- 或指定某个领域如：`工作方法论`、`教育方法论`、`人生复盘`

每条方法论归属于以下六大领域之一：

| 标签代号 | 领域 | 说明 |

| `#教育` | 教育 | 学习方法、知识管理、技能习得、阅读笔记 |
| `#人生` | 人生 | 价值观、人生阶段反思、重大决策、长期规划 |

## 工作流程

### 步骤 1：明确领域与主题

1. 与用户确认方法论所属的 **领域**（工作/生活/家庭/娱乐/教育/人生）
2. 确认方法论的 **主题名称**（用于文件命名）
3. 收集用户提供的经验、事件、思考素材

### 步骤 2：用黄金圈法则构建方法论

使用 [templates/methodology.md](templates/methodology.md) 模板，按以下结构组织内容：

#### Why — 为什么

- 这个方法论解决什么问题？
- 不这样做会怎样？
- 对你的人生/工作/成长意味着什么？

#### How — 怎么做

- 核心原则是什么？（3-5 条）
- 具体策略和步骤
- 关键注意事项和避坑指南

#### What — 具体行动

- 可落地的行动清单
- 成功标准 / 衡量指标
- 实际案例或经历佐证

### 步骤 3：打标签

每条方法论 **必须** 包含标签，用于后续检索：

```
tags: [#工作, #项目管理, #敏捷, #团队协作]
```

标签规则：
- **第一个标签**必须是六大领域之一
- 后续标签为细分主题标签，数量 2-5 个
- 标签使用简体中文
- 标签命名简洁，2-4 个字

### 步骤 4：更新索引

将新的方法论条目添加到 `index.md` 总索引文件中，包含：
- 标题、领域、标签
- 一句话摘要
- 文件链接

### 步骤 5：同步至 IMA 云笔记

> 通过 **ima-skill**（IMA OpenAPI）自动将方法论创建为 IMA 笔记。
> 前置条件：已按 `ima-skill` 的 Setup 步骤配置好 Client ID 和 API Key。

#### 5.1 凭证预检

调用 API 前先确认凭证可用（参考 `ima-skill` 的凭证预检流程）。
若凭证缺失，提示用户按 `ima-skill` Setup 步骤配置后重试。

#### 5.2 读取方法论文件内容

```bash
# 读取刚生成的方法论 Markdown 文件
content=$(cat .agent/methodology/{领域}/{method_name}.md)
```

> ⚠️ **UTF-8 编码强制要求**：写入 IMA 前必须确保内容为合法 UTF-8。
> 详见 `ima-skill` 的「UTF-8 编码强制要求」章节。
> 在 PowerShell 5.1 环境下，必须将请求 Body 转为 UTF-8 字节数组。

#### 5.3 创建 IMA 笔记

使用 `ima-skill` 的 `import_doc` 接口，将方法论内容作为新笔记创建：

```bash
# 新建到默认位置
ima_api "openapi/note/v1/import_doc" \
  '{"content_format": 1, "content": "'"$content"'"}'

# 或新建到指定「方法论」笔记本（需先用 list_note_folder_by_cursor 获取 folder_id）
ima_api "openapi/note/v1/import_doc" \
  '{"content_format": 1, "content": "'"$content"'", "folder_id": "方法论笔记本ID"}'
```

#### 5.4 验证上传结果

- API 返回 `doc_id` 表示创建成功
- 可用 `search_note_book` 按标题搜索确认笔记已存在
- 如已存在同名笔记且需要更新，使用 `append_doc` 追加差异内容

#### 5.5 笔记本管理建议

推荐在 IMA 中创建一个名为「方法论」的笔记本，按领域用标题前缀区分：

```
笔记标题格式：【{领域}】{方法论标题}
示例：【工作】向上管理方法论
      【教育】费曼学习法实践
```

## 质量标准

一条合格的方法论必须满足：

| 检查项 | 要求 |
|--------|------|
| Why 部分 | 至少说明 1 个核心动机和 1 个不行动的后果 |
| How 部分 | 至少包含 3 条可执行的原则或步骤 |
| What 部分 | 至少包含 1 个具体行动项和 1 个成功标准 |
| 标签 | 第一个标签为领域标签，总数 3-6 个 |
| 可读性 | 他人阅读后能理解并执行 |
| 独特性 | 包含个人经历或独特见解，不是泛泛而谈 |

## 反模式

| ❌ 不要这样做 | ✅ 应该这样做 |
|--------------|-------------|
| 只写 What（流水账） | 先思考 Why，再推导 How 和 What |
| 标签太笼统（只写 `#工作`） | 加上细分标签（`#工作 #沟通 #向上管理`） |
| 方法论停在理论层面 | 必须有具体可落地的行动清单 |
| 一次写太多方法论 | 每次专注一条，写深写透 |
| 写完就忘 | 定期回顾和更新 index.md |
| 只截图不写入文件 | 所有内容必须持久化到 Markdown 文件 |
